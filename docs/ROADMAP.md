**ROADMAP**

**Grassroots Procurement Transparency & Supplier Support Initiative**

*Zimbabwe · Procurement (PRAZ-adjacent) · Civic Tech*

*This roadmap sequences the idea from a zero-permission, zero-legal-risk
starting point through to a possible formal partnership with the
Procurement Regulatory Authority of Zimbabwe (PRAZ). It is deliberately
grassroots-first: every phase is designed to prove real demand and
generate real evidence before any institutional dependency is
introduced.*

**Guiding Principles**

-   Build for suppliers and citizens first; treat PRAZ as a partner to
    earn, not a gatekeeper to seek permission from on day one.

-   No bulk scraping or automated republishing of PRAZ data until a
    partnership or explicit data-sharing agreement exists --- PRAZ\'s
    Terms & Conditions assign ownership of procurement Information to
    PRAZ and issuing agencies.

-   Start human-in-the-loop: manually sourced or user-submitted content
    only, until trust and evidence justify automation.

-   Frame everything as extending Zimbabwe\'s existing reform mandate
    (PRAZ\'s own transparency goals; the national Citizen Engagement and
    Scoring Platform launched June 2026) rather than criticizing it.

-   Each phase has a clear, small exit test --- do not advance on
    ambition, advance on evidence.

**Phase Overview**

+-----------------------------------------------------------------------+
| **PHASE 0 --- Validate the Problem**                                  |
|                                                                       |
| *Weeks 1--3*                                                          |
+-----------------------------------------------------------------------+
| **Goal:** Confirm suppliers actually experience the pain points       |
| assumed (tender discovery, plain-language barriers, missed deadlines) |
| before writing product code.                                          |
+-----------------------------------------------------------------------+
| **Actions:**                                                          |
|                                                                       |
| -   Identify and speak with 8--12 real suppliers (via personal        |
|     network, ABMA/developer contacts, or one business association     |
|     contact).                                                         |
|                                                                       |
| -   Ask open questions: how do you currently find tenders, what\'s    |
|     confusing, what have you missed or lost bids over.                |
|                                                                       |
| -   Manually review PRAZ eGP tender notices to understand real        |
|     document complexity and language.                                 |
|                                                                       |
| -   Look for the local Follow The Money chapter (Magamba Network) and |
|     Open Parly ZW --- hold an exploratory conversation, not a pitch.  |
+-----------------------------------------------------------------------+
| **Exit criteria (what unlocks the next phase):** You can name the top |
| 2--3 specific pain points in suppliers\' own words, and at least 5    |
| suppliers say they would try a free tool that addresses one of them.  |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **PHASE 1 --- WhatsApp Supplier Assistant (MVP)**                     |
|                                                                       |
| *Weeks 3--8*                                                          |
+-----------------------------------------------------------------------+
| **Goal:** Ship the smallest useful tool: a WhatsApp-based assistant   |
| that turns tender notices suppliers already have into plain-language  |
| summaries and reminders --- no scraping, no bulk data.                |
+-----------------------------------------------------------------------+
| **Core features:**                                                    |
|                                                                       |
| -   Supplier forwards or pastes a tender notice they found            |
|     themselves.                                                       |
|                                                                       |
| -   AI-generated plain-language summary: what\'s required, deadline,  |
|     category, eligibility.                                            |
|                                                                       |
| -   Document checklist generation.                                    |
|                                                                       |
| -   Manual deadline reminder set by the supplier.                     |
|                                                                       |
| **Pilot group:** 5--10 suppliers sourced from Phase 0 conversations.  |
|                                                                       |
| **Legal posture:** Zero data-ownership risk --- you only process      |
| content the user voluntarily shares with you, the same as any         |
| assistant or consultant would.                                        |
+-----------------------------------------------------------------------+
| **Exit criteria (what unlocks the next phase):** At least 5 pilot     |
| suppliers use it more than once without prompting, and can describe a |
| concrete way it helped (e.g. \"caught a deadline I would have         |
| missed\").                                                            |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **PHASE 2 --- Citizen & Supplier Tip-Line (Verification Layer)**      |
|                                                                       |
| *Months 2--4, overlapping Phase 1*                                    |
+-----------------------------------------------------------------------+
| **Goal:** Introduce the Ojoconmipisto-style mechanic: citizens or     |
| suppliers report a concern (stalled contract, undelivered project,    |
| irregular award), and it is manually cross-checked against PRAZ\'s    |
| own public award notices.                                             |
+-----------------------------------------------------------------------+
| **Core features:**                                                    |
|                                                                       |
| -   Same WhatsApp channel accepts tip submissions in addition to      |
|     supplier requests.                                                |
|                                                                       |
| -   Manual (human-in-the-loop) verification against publicly viewable |
|     PRAZ award notices --- one at a time, not scraped in bulk.        |
|                                                                       |
| -   Verified findings published narratively (case-by-case), not yet   |
|     as a scored dataset.                                              |
|                                                                       |
| **Why this order matters:** Trust and a track record of accuracy must |
| exist before anything resembling a public rating system is credible   |
| or legally defensible.                                                |
+-----------------------------------------------------------------------+
| **Exit criteria (what unlocks the next phase):** A handful of         |
| verified, published cases exist with no factual disputes or           |
| correction requests, and at least one has been referenced by a        |
| supplier, journalist, or local organisation.                          |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **PHASE 3 --- PRAZ Engagement**                                       |
|                                                                       |
| *Months 4--7*                                                         |
+-----------------------------------------------------------------------+
| **Goal:** Approach PRAZ from a position of evidence, not speculation  |
| --- proposing a scoped, time-limited pilot rather than broad data     |
| access.                                                               |
+-----------------------------------------------------------------------+
| **Actions:**                                                          |
|                                                                       |
| -   Send a short concept note to feedback@praz.org.zw, referencing    |
|     the working prototype and citing the national Citizen Engagement  |
|     and Scoring Platform (launched June 2026) as precedent for        |
|     citizen-facing accountability tools.                              |
|                                                                       |
| -   Propose a narrow, bounded pilot: e.g. alerts and plain-language   |
|     summaries for one procurement category, for 60--90 days.          |
|                                                                       |
| -   Follow up by phone after \~1 week if no response.                 |
|                                                                       |
| -   Expect a multi-month institutional process; do not block other    |
|     work on this.                                                     |
+-----------------------------------------------------------------------+
| **Exit criteria (what unlocks the next phase):** PRAZ agrees to a     |
| scoped pilot, a data-sharing conversation, or at minimum a meeting    |
| --- or, if it stalls, the tool continues to operate and grow on       |
| publicly available, manually-sourced information alone.               |
+-----------------------------------------------------------------------+

+-----------------------------------------------------------------------+
| **PHASE 4 --- Structured Data & Scoring Layer**                       |
|                                                                       |
| *Months 7--12+, conditional on Phase 2--3 evidence base*              |
+-----------------------------------------------------------------------+
| **Goal:** Only once a real base of verified cases exists --- and      |
| ideally with PRAZ\'s cooperation or at least awareness --- introduce  |
| an aggregated scoring or scorecard layer (e.g. supplier delivery      |
| reliability, agency tender-cancellation patterns).                    |
+-----------------------------------------------------------------------+
| **Prerequisites before building this:**                               |
|                                                                       |
| -   A defined evidence/correction process (how a flagged supplier or  |
|     agency can respond or dispute).                                   |
|                                                                       |
| -   Legal review of defamation and reputational-risk exposure         |
|     specific to Zimbabwean law.                                       |
|                                                                       |
| -   A meaningful volume of verified cases underneath the score ---    |
|     not a cold-start rating system.                                   |
+-----------------------------------------------------------------------+
| **Exit criteria (what unlocks the next phase):** This phase is        |
| intentionally the last, and should not be attempted opportunistically |
| ahead of its prerequisites, regardless of funder interest.            |
+-----------------------------------------------------------------------+

**Parallel Track: Funding & Positioning**

This runs alongside the phases above, not before them.

-   Early (Phase 0--1): document everything --- conversations, pilot
    usage, verified cases --- as evidence for later funding
    applications.

-   Mid (Phase 2--3): approach Mozilla Builders, Shuttleworth
    Foundation, Indigo Trust, or the African Union Civic Tech Fund with
    a working prototype and real user evidence, not a concept alone.

-   Position consistently as a three-way model --- citizens/suppliers,
    civil society, and government --- the pattern shared by the most
    durable precedents (ProZorro, IFollowTheMoney), rather than a tool
    built in opposition to PRAZ.

-   Consider reaching out to Magamba Network (local Follow The Money /
    Open Parly ZW) as a potential collaborator or advisor, not just a
    precedent to study.

**What This Roadmap Deliberately Avoids (For Now)**

-   Automated scraping or bulk republishing of PRAZ eGP data.

-   Any public scoring or reputational claims about named suppliers or
    agencies before Phase 4.

-   Dependency on PRAZ approval to begin building or piloting.

-   Full CivicPass-style scope (citizen services, payments, identity,
    national procurement platform) --- this roadmap deliberately narrows
    to the procurement-transparency and supplier-support wedge only.
